# From Script to Tool: Building Robust, Meaningful Tools with PowerShell

I have been asked the same question many times over the years and it usually goes something like this: “How do I know when to do something through the command line, a script, a function or a module for something in PowerShell?”

The short answer, is “YES”. But the deeper question is, “What kind of Tool are you trying to build?”

When we think of “Tools”, they can come in all sorts of shapes, sizes and uses. PowerShell tools are no different and there are usually no wrong answers here.

In this session, we will explore the evolution of building tools in PowerShell. From trial and error at the command line, to crafting a complete module. I will use some examples from my own experience when it comes to creating re-usable tools with our favorite scripting language and we will watch how a small one liner can evolve into a tool that you and your colleagues can’t remember how you lived without.
